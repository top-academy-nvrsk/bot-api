# bot-api
репозиторий для апишки бота

## что делать
для начала нужно как минимум иметь установленный гит на компе. проверить можно просто: `git --version`. если выпало что-то отличное от примера ниже, его надо установить с официального сайта: https://git-scm.com/
```bash
C:\Users\user>git --version
git version 2.45.1.windows.1
```

### основы гита
все операции с гитом проводятся в основном через консоль, но если ты работаешь в vscode, там есть расширения. однако я все-таки рекомендую сначала освоить именно консольный гит, т.к. он дает большую прозрачность в вопросе того, что ты делаешь
`git clone`: склонировать репозиторий для локальной работы над ним. в нашем случае нужно будет вводить `git clone https://github.com/top-academy-nvrsk/bot-api`
клонировать репозиторий заново после внесений в main необязательно, так что если таковые были внесены, нужно писать `git pull origin main`
`git checkout`: переключиться на ветку. чтобы просмотреть ветки, нужно ввести ``
`git add .`: добавить все файлы в данной папке в коммит (коммитом называют снимок состояния проекта в определенный момент времени)
`git commit -m 'message'`: коммитнуть изменения и добавить к коммиту сообщение (писать их ОБЯЗАТЕЛЬНО)
`git push`: отправить изменения в удаленный репозиторий. в данном случае обязательно нужно писать `git push -u origin <название твоей ветки>`, чтобы случайно не закоммитить в main-ветку (если это произойдет, буду бить и очень больно)

в принципе, больше знать пока ничего и не надо. разве что упомяну, что файл .gitignore трогать НЕ надо, дабы в репозитории не оказалось лишнего мусора
